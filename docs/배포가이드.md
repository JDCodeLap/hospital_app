# 🚀 배포 가이드 (Vercel + Railway)

> 만든 앱을 인터넷에 올려 **남들도 주소로 접속해 쓰게** 하는 과정이에요.
> 이 앱은 부품이 3개라 따로 올립니다: **화면(Vercel) · 서버(Railway) · 데이터베이스(이미 Supabase)**.
> 비개발자 기준으로, **클릭 순서 그대로** 적었어요. 위에서부터 차례대로 따라 하시면 됩니다.

---

## ⏱️ 시작 전 준비물
- **GitHub 계정** (이미 있음 — `JDCodeLap/hospital_app` 저장소)
- **`backend/.env` 파일의 값 2개** (내 PC에만 있고 GitHub엔 안 올라가요 — 그래서 직접 복사해 넣어야 함):
  - `DATABASE_URL` (Supabase 주소)
  - `JWT_SECRET_KEY` (로그인 토큰 비밀키)
  - 이 두 값을 메모장에 잠깐 복사해 두세요. (CORS는 아래서 만듦)
- 💳 Railway는 체험 크레딧($5) 소진 후 **월 $5~ 유료**예요. 계속 쓰려면 결제수단 등록이 필요해요.

> ⚠️ **순서가 중요해요: 서버(Railway) 먼저 → 화면(Vercel) 나중 → 마지막에 둘을 연결.**
> 화면이 "어느 서버에 말 걸지" 알아야 하고, 서버는 "어느 화면의 요청을 허용할지" 알아야 하거든요.

---

## 1단계. 서버(백엔드)를 Railway에 올리기

1. **[railway.app](https://railway.app)** 접속 → **Login** → **GitHub 계정으로 로그인**.
2. **New Project** → **Deploy from GitHub repo** → 저장소 목록에서 **`JDCodeLap/hospital_app`** 선택.
   - 권한 요청이 뜨면 저장소 접근을 허용해줘요.
3. 서비스가 하나 생기면, 그 서비스를 클릭 → **Settings(설정)** 탭으로 가요.
   - **Root Directory(루트 폴더)** 항목에 **`backend`** 입력. ⭐ (서버 코드가 `backend/` 폴더 안에 있어서 꼭 필요해요)
   - 저장하면 `backend/railway.json`을 Railway가 읽어 **빌드·시작 명령·헬스체크**를 자동 적용해요.
4. **Variables(환경변수)** 탭 → **New Variable**로 아래 3개를 추가:
   | 이름 | 값 |
   |------|----|
   | `DATABASE_URL` | (메모해 둔 Supabase 주소 그대로 붙여넣기) |
   | `JWT_SECRET_KEY` | (메모해 둔 비밀키 그대로 붙여넣기) |
   | `CORS_ORIGINS` | 일단 `http://localhost:3000` 입력 (화면 주소 나오면 3단계에서 교체) |
5. **Settings → Networking(네트워킹)** → **Generate Domain(도메인 만들기)** 클릭. ⭐
   - 이걸 눌러야 **바깥에서 접속할 공개 주소**가 생겨요. 예: **`https://hospital-app-api.up.railway.app`**
6. 잠시 기다려 배포가 끝나면(초록불), 그 주소 뒤에 **`/api/health`** 를 붙여 브라우저로 열어보세요.
   - `{"status":"ok"}` 가 보이면 서버 성공! ✅
   - **이 서버 주소를 메모해 두세요.** (2단계에서 화면에 알려줄 주소)

---

## 2단계. 화면(프론트)을 Vercel에 올리기

1. **[vercel.com](https://vercel.com)** 접속 → **로그인** → **GitHub 계정으로 로그인**.
2. **Add New... → Project** → 저장소 목록에서 **`JDCodeLap/hospital_app`** → **Import**.
3. ⭐ **Root Directory(루트 폴더)** 를 **`frontend`** 로 바꿔요. ("Edit" 눌러 `frontend` 선택 — 화면 코드가 `frontend/` 안에 있어요)
4. **Environment Variables(환경변수)** 에 1개 추가:
   | 이름 | 값 |
   |------|----|
   | `NEXT_PUBLIC_API_BASE_URL` | (1단계에서 메모한 **Railway 서버 주소**, 예: `https://hospital-app-api.up.railway.app`) |
5. **Deploy** 클릭 → 1~2분 뒤 배포 완료.
6. 만들어진 화면 주소를 확인해요. 예: **`https://hospital-app.vercel.app`**
   - **이 화면 주소를 메모해 두세요.** (3단계에서 서버에 알려줄 주소)

---

## 3단계. 서버와 화면을 서로 연결 (CORS)

지금은 서버가 "이 화면의 요청을 받아도 되나?"를 몰라서 막아요. 화면 주소를 서버에 등록해줘야 통해요.

1. **Railway 대시보드** → 서버 서비스 → **Variables** 탭.
2. **`CORS_ORIGINS`** 값을 **2단계에서 메모한 화면 주소**로 바꿔요. 예: `https://hospital-app.vercel.app`
   - (주소 끝에 슬래시 `/` 는 빼고 넣어요)
3. 저장하면 서버가 **자동으로 다시 배포**돼요(1~2분).

---

## 4단계. 진짜 되는지 확인 🎉

1. **2단계 화면 주소**(`https://....vercel.app`)를 브라우저로 열기.
2. 로그인 화면이 뜨면 데모 계정으로 로그인:
   - 간호사: `nurse1` / `test1234`
   - 관리자: `admin1` / `admin1234`
3. 환자 목록·등록·수정이 다 되면 배포 성공! 휴대폰으로도 같은 주소로 접속돼요. 📱

---

## ❓ 자주 막히는 곳
- **공개 주소가 안 보여요** → 1단계 5번 **Generate Domain**을 눌렀는지 확인. 이걸 눌러야 주소가 생겨요.
- **"백엔드에 연결할 수 없습니다"** → ① Vercel의 `NEXT_PUBLIC_API_BASE_URL`이 Railway 주소와 정확히 같은지 ② Railway의 `CORS_ORIGINS`가 Vercel 주소와 정확히 같은지(오타·끝 슬래시 주의) 확인.
- **빌드 실패** → Railway **Settings의 Root Directory가 `backend`** 인지 확인.
- **환경변수 바꾼 게 반영 안 돼요** → 바꾼 뒤 **다시 배포(Redeploy)** 가 끝나야 적용돼요.
- **비밀값은 절대 코드/문서에 적지 않기** → 이 저장소는 공개라, DB 주소·비밀키는 항상 Railway·Vercel **대시보드에만** 넣어요.

## 🔁 앞으로 코드를 고치면?
