# 병원 통합 안전 관리 앱 — EXPERIENCE.md

## Foundation (기본 전제)

- **사용 환경:** 휴대폰 앱 + 컴퓨터(웹) 둘 다. 같은 기능을 두 화면 크기에 맞춰 보여준다.
- **사용자:** 중간 규모 병원의 모든 직군(의사·간호사·원무과). 1차는 직군 구분 없이 동일 화면, 로그인한 직원이면 열람 가능.
- **시각 정체성:** `DESIGN.md` 참조 (차분한 다크 모드).
- **설계 원칙:** 바쁜 의료 현장 → 클릭 적게, 글자 크게, 중요 정보는 위로.

## Information Architecture (화면 구조)

```
로그인
 └─ 홈 대시보드 (시작 화면)
     ├─ 내 환자 목록
     ├─ 급한 알림 요약
     ├─ [환자 검색] → 환자 통합 화면
     ├─ [알림] → 알림 모아보기
     └─ [환자 흐름판] → 전체 환자 단계 현황
 환자 통합 화면 (⭐ 핵심, 어디서든 진입)
     └─ 기본정보 · 방문기록 · 진단 · 투약 · 검사결과 · 수납 · 안전경고
```

**화면 목록**
1. **로그인** — 직원 인증 (간단한 ID/비밀번호 수준, 1차)
2. **홈 대시보드** — 앱 시작점. 내 환자 + 급한 알림 한눈에.
3. **환자 검색** — 이름/등록번호로 빠르게 찾기 `{FR2}`
4. **환자 통합 화면** ⭐ — 한 환자의 모든 정보 + 안전 경고 `{FR1,FR3,FR4,FR5,FR6,FR7}`
5. **알림 모아보기** — 놓치면 안 되는 것 전체 목록 `{FR4-7,FR10}`
6. **환자 흐름판** — 모든 환자가 지금 어느 단계인지 `{FR8,FR9,FR10}`

> IA 닫힘 확인: PRD의 모든 기능(FR1~10)이 위 6개 화면 중 하나에 들어감. ✅

## Voice and Tone (말투/문구)

- **간결·단정적.** 바쁜 현장이므로 군더더기 없이. "페니실린 알레르기 — 처방 주의" 처럼 핵심만.
- **경고는 명령형으로 명확하게.** "확인 필요", "다음 단계로", "투약 시간입니다".
- 불필요한 친근함·이모지 남용 금지. 단, 위험/주의/완료는 아이콘으로 빠른 인지 보조(⚠️ 🔔 ✅).

## Component Patterns (구성 요소 동작)

- **환자 카드:** 누르면 → 환자 통합 화면으로. 카드에 이름·나이·현재 단계·경고 배지 표시.
- **안전 경고 배너:** 환자에게 위험 정보가 있으면 통합 화면 맨 위에 빨간 배너. 누르면 상세.
- **알림 배지:** 상단 🔔 아이콘에 안 읽은 알림 개수를 빨간 점/숫자로.
- **단계 표시 바:** 접수→진료→검사→수납을 가로 막대로, 현재 위치 하늘색 강조 `{accent.primary}`.

## State Patterns (상태 표현)

- **로딩:** 정보 불러오는 중 → 자리표시(스켈레톤) 회색 박스.
- **빈 상태:** 환자/알림 없을 때 → "표시할 항목이 없습니다" 안내.
- **위험 상태:** 알레르기·금기 → 빨강 `{danger}` 배너 + 아이콘.
- **주의 상태:** 투약 시간·대기 초과 → 앰버 `{warning}`.
- **완료 상태:** 절차 체크 완료·수납 완료 → 초록 `{success}`.
- **놓침 방지:** 안전 경고는 직원이 "확인" 누르기 전까지 사라지지 않음 `{FR7}`.

## Interaction Primitives (기본 조작)

- **탭/클릭:** 카드·버튼 누르기 (기본).
- **검색:** 상단 검색창에 입력 → 실시간 필터.
- **체크리스트:** 항목 탭하면 체크 ✅. 전부 체크돼야 "다음 단계" 버튼 활성화 `{FR6}`.
- **알림 확인:** 알림을 탭해서 "확인/조치" → 목록에서 정리됨.

## Accessibility Floor (접근성 최소 기준)

- 터치 영역 최소 44×44px (장갑 낀 손, 바쁜 손 고려).
- 다크 배경 위 글자 명도 대비 충분히 (보조 글자도 읽히게).
- 색만으로 정보 전달 금지 → 위험은 색 + 아이콘 + 글자 3중 표시.
- 글자 크기 키울 수 있게(시스템 글꼴 크기 존중).

## Key Flows (핵심 사용 흐름)

### Flow 1 — 간호사 지영, 위험한 처방을 막다 (⭐ 클라이맥스)
> 지영은 야간 근무 간호사. 환자 김철수님 처방을 입력하려 한다.

1. 지영이 홈 대시보드에서 "김철수" 환자 카드를 누른다.
2. 환자 통합 화면 맨 위에 **빨간 경고 배너**: "⚠️ 페니실린 알레르기".
3. 의사가 페니실린 계열 약을 처방 입력하는 순간, **화면이 멈추고 경고 팝업**이 뜬다. `{FR4}`
4. **(클라이맥스)** "이 환자는 페니실린 알레르기입니다. 그래도 처방하시겠습니까?" → 지영이 한 번 더 확인하고 다른 약으로 변경.
5. 사고가 날 뻔한 순간을, 앱이 먼저 막았다. ✅

### Flow 2 — 원무과 민수, 환자 흐름을 한눈에 본다
1. 민수가 홈에서 "환자 흐름판"을 연다.
2. 모든 환자가 접수→진료→검사→수납 어느 단계인지 막대로 보인다. `{FR9}`
3. 한 환자가 검사 단계에서 1시간째 멈춰 있다 → **앰버 색으로 깜빡** + 알림. `{FR10}`
4. 민수가 해당 환자를 눌러 상황 확인 후 다음 안내를 진행한다.

### Flow 3 — 투약 시간 알림
1. 201호 환자 투약 시간이 됨 → 담당 간호사 휴대폰에 🔔 알림. `{FR5}`
2. 알림을 누르면 환자 통합 화면의 투약 정보로 바로 이동.
3. 투약 완료 체크 → 초록 `{success}` 표시, 알림 정리됨.

## Responsive & Platform (화면 크기 대응)

- **휴대폼(좁은 화면):** 카드 세로 한 줄 쌓기. 하단 탭바(홈/검색/알림/흐름판).
- **웹(넓은 화면):** 왼쪽 세로 메뉴 + 오른쪽 콘텐츠. 환자 통합 화면은 2~3단 배치로 정보를 더 펼침.
- 두 화면 모두 같은 기능, 배치만 다름.
