# Story 3.1: 알레르기/금기 경고

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 병원 직원,
I want 환자에게 위험한 약(알레르기/금기 약물)을 처방 입력하려 할 때 즉시 경고를 받고 한 번 더 확인하기를,
so that 알레르기 사고를 막는다. (FR4)

## Acceptance Criteria

epics.md 원문(Story 3.1) + EXPERIENCE.md(Flow ③ "페니실린 계열 처방 순간 화면 멈추고 경고 팝업") + 2.4(쓰기→broadcast 파이프)·2.1(allergies 필드)·1.3/1.4(토큰 인증)에서 도출한 검증 가능한 항목입니다.

1. **(원문)** 환자에게 알레르기/금기 정보가 등록된 상태에서, **충돌하는 약을 처방 입력**하면 **빨간 경고 팝업**이 떠서 "이 환자는 OO 알레르기입니다"라고 알린다. **그리고 계속할지 한 번 더 확인**을 요구한다(취소 / 계속).
2. 백엔드에 **처방(투약) 추가 엔드포인트** `POST /api/patients/{patient_id}/medications`(보호됨, `get_current_user`)를 추가한다. 본문: `drug_name`(필수), `dose`(선택), `schedule`(선택), `acknowledged`(기본 false). 환자 없으면 404.
3. **충돌 판정 로직**: 환자의 `allergies`(콤마 구분)와 처방하려는 `drug_name`을 비교해 충돌하는 알레르기 항목 목록을 만든다. **직접 일치**(알레르기 명이 약 이름에 포함, 대소문자 무시)뿐 아니라 **계열(class) 매칭**도 한다 — 예: `페니실린` 알레르기 + `아목시실린`(페니실린계) 처방 → 충돌. (EXPERIENCE.md Flow ③ "페니실린 **계열**" 요구 반영)
4. **충돌이 있고 `acknowledged=false`면 저장하지 않고 409**(Conflict)로 응답한다 — 본문에 충돌 항목(`conflicts: ["페니실린"]`)과 약 이름을 담아 프론트 팝업이 "이 환자는 페니실린 알레르기입니다"를 만들 수 있게 한다. **백엔드가 안전장치를 강제**(프론트만 믿지 않음) — 확인 없이는 위험 약이 저장되지 않는다.
5. **충돌이 없거나(`conflicts` 비었거나)** **`acknowledged=true`(직원이 위험을 확인하고 계속 선택)**면 투약(`Medication`, `status="active"`)을 저장하고, **2.4의 `manager.broadcast(patient_id, {"type":"patient_updated", ...})`를 그대로 호출**한다(저장 결과 반환). → 그 환자 화면을 보는 클라이언트는 새로고침 없이 갱신되어 새 처방이 투약 카드에 나타난다.
6. 프론트 `PatientDetail`(2.3/2.4)에 **처방 입력 폼**(약 이름·용량·시간 + "처방 추가" 버튼)을 더한다. 제출 시:
   - **200** → 성공(폼 비움). WS 갱신으로 투약 카드에 새 처방 표시(또는 즉시 reload).
   - **409** → 응답의 `conflicts`로 **빨간 경고 팝업**(`role="alertdialog"`)을 띄움: "⚠ 이 환자는 OO 알레르기입니다 — 계속하시겠습니까?" + [취소] [계속 처방].
     - **취소** → 팝업 닫고 저장 안 함.
     - **계속 처방** → 같은 내용 + `acknowledged: true`로 **다시 POST** → 200이면 저장·갱신.
   - **401/403** → `clearToken()` + `/login`(기존 정책 일관). 그 외 오류 → 폼 근처에 인라인 오류 문구(화면 전체를 깨지 않음).
7. **2.3/2.4 화면 동작을 깨지 않는다** — 기존 카드 레이아웃·로딩/404/401/오류 상태·AuthGuard 보호·클라이언트 fetch(1.4)·WebSocket 구독/조용한 갱신(2.4) 모두 유지. 처방 폼·경고 팝업은 그 위에 얹는다.

> **범위 밖(하지 말 것):** 안전 배너가 "확인 전까지 안 사라지게" 유지하는 것 = **Story 3.2**(지금은 2.3의 정적 배너 그대로 둠). / 투약 시간 알림 = 3.3 / 필수 절차 체크리스트 = 3.4 / 투약 **수정·삭제** 및 전체 CRUD, 용량 단위 검증, **실제 약물-약물 상호작용(DDI) 엔진/외부 약품 DB** = 범위 밖(우리 충돌 판정은 **알레르기명 직접 + 작은 계열 매핑**까지만). / 역할별 권한(처방 권한 구분) = Epic 5. / 처방 입력은 **이 화면 한 곳**에서만(별도 처방 페이지·검색 연동 후속).

## Dev Notes

### 이 스토리의 핵심 (한 줄)
환자 통합 화면에 **처방 입력**을 추가하고, 그 처방이 **환자 알레르기/금기와 충돌하면 빨간 팝업으로 경고 + 한 번 더 확인**을 받는다. 안전장치는 **백엔드가 강제**(충돌 시 확인 없으면 409로 저장 거부)하고, 확인하면 저장 후 **2.4의 실시간 파이프(broadcast)** 로 화면이 자동 갱신된다.

### ⚠️ 이 스토리의 가장 중요한 설계 판단
- **안전 검사는 백엔드에서 강제한다(프론트만 믿지 않음).** 프론트 팝업은 UX(경고 보여주기)지만, 실제 "확인 없이는 위험 약 저장 불가"는 백엔드 409로 보장. 프론트를 우회해도(예: /docs 직접 호출) `acknowledged` 없이는 충돌 약이 저장되지 않는다. → FR4(안전)의 핵심.
- **"확인하면 계속 가능"** = 같은 요청에 `acknowledged: true`를 실어 재전송. 백엔드는 이때 저장을 허용(직원이 책임지고 진행한 것으로 간주). 이게 AC의 "계속할지 한 번 더 확인".
- **충돌 판정은 단순하게(데모 수준).** 알레르기명 **직접 부분일치** + **작은 계열 매핑**(`safety.py`)까지만. 진짜 약물 상호작용 엔진/외부 약품 DB는 범위 밖. 그래도 EXPERIENCE.md가 요구하는 "페니실린 **계열**(아목시실린 등)" 시나리오는 매핑으로 충족.
- **저장 후 화면 갱신은 새로 만들지 않는다.** 2.4가 깔아둔 `manager.broadcast` + `PatientDetail`의 WS 구독이 이미 "이 환자 바뀜 → 조용히 reload"를 처리한다. 처방 저장 시 broadcast만 호출하면 투약 카드가 자동 갱신됨. (WS 미연결 대비, 성공 후 직접 reload 한 번도 무방)

### 2.4에서 이어받는 사실 — 반드시 인지(보존/재사용)
- ✅ **실시간 파이프 그대로 재사용**: `backend/app/realtime.py`의 전역 `manager`와 `manager.broadcast(patient_id, message)`. 처방 저장 후 이 한 줄만 호출하면 끝. [Source: backend/app/realtime.py, main.py(add_visit 패턴)]
- ✅ **쓰기 엔드포인트 패턴**: `POST /api/patients/{patient_id}/visits`(`add_visit`)가 이미 **`async def` + `Depends(get_current_user)` + 404 + commit + `await manager.broadcast`** 형태. 처방 엔드포인트는 이 패턴을 그대로 따르고 **충돌 판정만 추가**. [Source: backend/app/main.py:399-425]
- ✅ **프론트 WS 구독/조용한 갱신**: `PatientDetail`의 `loadBundle(signal, silent)`와 WS effect가 `patient_updated` 신호 시 `loadBundle(undefined, true)`로 조용히 갱신. **이 로직을 건드리지 말 것** — 처방 폼은 그 위에 얹기만. [Source: frontend/src/components/PatientDetail.tsx:79-180]

### 확정 기술/패턴 가이드

- **충돌 판정 헬퍼(요지) — `backend/app/safety.py`(NEW):**
  ```python
  # 알레르기 항목 → 충돌하는 약 이름 키워드(계열). 직접 일치는 매핑이 없어도 항상 검사한다.
  CONTRAINDICATION_MAP: dict[str, list[str]] = {
      "페니실린": ["페니실린", "아목시실린", "암피실린", "오구멘틴", "피페라실린"],
      "아스피린": ["아스피린", "아세틸살리실산"],
      "설파제": ["설파", "박트림", "설파메톡사졸"],
      "조영제": ["조영제"],
  }

  def check_contraindications(allergies: str, drug_name: str) -> list[str]:
      """환자 알레르기와 처방 약 이름을 비교해 '충돌한 알레르기 항목' 목록을 돌려준다."""
      drug = (drug_name or "").strip().lower()
      if not drug:
          return []
      hits: list[str] = []
      for raw in (allergies or "").split(","):
          term = raw.strip()
          if not term:
              continue
          low = term.lower()
          conflict = low in drug  # 직접 부분일치(알레르기명이 약 이름에 들어감)
          if not conflict:
              for kw in CONTRAINDICATION_MAP.get(term, []):  # 계열 매칭
                  if kw.lower() in drug:
                      conflict = True
                      break
          if conflict and term not in hits:
              hits.append(term)
      return hits
  ```
- **처방 엔드포인트(요지) — `main.py`:**
  ```python
  from pydantic import BaseModel
  from .safety import check_contraindications

  class MedicationIn(BaseModel):
      drug_name: str
      dose: str = ""
      schedule: str = ""
      acknowledged: bool = False  # 경고를 보고도 계속 진행할 때 true

  @app.post("/api/patients/{patient_id}/medications")
  async def add_medication(
      patient_id: int,
      payload: MedicationIn,
      current: Staff = Depends(get_current_user),
      session: Session = Depends(get_session),
  ) -> dict[str, object]:
      patient = session.get(Patient, patient_id)
      if patient is None:
          raise HTTPException(status_code=404, detail="환자를 찾을 수 없습니다")
      conflicts = check_contraindications(patient.allergies or "", payload.drug_name)
      if conflicts and not payload.acknowledged:
          # 저장하지 않고 경고 — 프론트가 팝업을 만들 정보를 함께 전달
          raise HTTPException(
              status_code=409,
              detail={"conflicts": conflicts, "drug_name": payload.drug_name,
                      "message": f"이 환자는 {', '.join(conflicts)} 알레르기입니다"},
          )
      med = Medication(patient_id=patient_id, drug_name=payload.drug_name,
                       dose=payload.dose, schedule=payload.schedule, status="active")
      session.add(med); session.commit(); session.refresh(med)
      await manager.broadcast(patient_id, {"type": "patient_updated", "patient_id": patient_id})
      return {"id": med.id, "patient_id": patient_id, "drug_name": med.drug_name}
  ```

### 흔한 실수 방지
- ❌ 안전 검사를 프론트에서만 함 → 우회 가능. **백엔드 409로 강제**(프론트 팝업은 UX). 둘 다 둔다.
- ❌ 충돌인데도 그냥 저장 → 반드시 `acknowledged` 없으면 **저장 전에** 409. (저장 후 경고는 의미 없음)
- ❌ 계열 매칭 누락(직접 일치만) → 김철수 + `아목시실린`이 안 걸림. EXPERIENCE.md는 "페니실린 **계열**" 요구 → `CONTRAINDICATION_MAP` 필수.
- ❌ `async def` 안 씀 → `await manager.broadcast` 동작 안 함(2.4 교훈). 처방 엔드포인트는 **async**.
- ❌ FastAPI의 `HTTPException(detail=dict)` 응답 모양 착각 → 프론트는 `body.detail.conflicts`로 읽어야 함(detail 안에 dict가 들어감).
- ❌ 2.3/2.4의 fetch·WS·카드 로직을 갈아엎음 → **추가만** 하고 기존 동작 보존(AC #7).
- ❌ 처방 폼 effect에서 동기 setState 직접 → lint `react-hooks/set-state-in-effect`(1.4/2.x 교훈). 제출은 이벤트 핸들러라 보통 안전하지만, 초기화 로직을 effect에 두지 말 것.
- ❌ 안전 배너를 "확인 전 유지"로 바꾸려 함 → 그건 **3.2 범위**. 3.1은 정적 배너 그대로.
