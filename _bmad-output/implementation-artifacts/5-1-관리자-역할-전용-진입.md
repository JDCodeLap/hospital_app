# Story 5.1: 관리자 역할 & 전용 진입

Status: done

## Story

As a 병원 관리자,
I want 관리자만 들어갈 수 있는 별도 영역(관리자 페이지)을,
So that 일반 직원은 관리 기능을 못 건드린다. (NFR2)

## Acceptance Criteria

1. 직원 계정에 **"관리자(admin)" 역할**을 지정할 수 있다 — `Staff.role`에 `"admin"` 값을 둔다(기존은 전원 `"staff"`). **관리자 시드 계정**(예: `admin1`/`admin1234`)이 있다.
2. **백엔드 관리자 전용 보호 장치** `get_current_admin`(의존성)이 있다 — 로그인했어도 `role != "admin"`이면 **403으로 거부**(진짜 자물쇠는 백엔드, 프론트 숨김에만 의존하지 않음).
3. **관리자 전용 화면 `/admin`**이 있다 — 관리자만 보이는 진입점. 일반 직원이 직접 URL로 들어가도 **차단**된다(관리자 아니면 홈으로 돌려보냄).
4. **네비게이션(AppShell)에 "관리자" 메뉴**가 **관리자에게만** 보인다 — 일반 직원에게는 그 메뉴가 보이지 않는다.
5. `/admin` 화면은 **관리자 페이지 시작점**(랜딩) — 앞으로 추가될 기능(직원 관리 5.2 / 권한 5.3 / 현황 5.4 / 기준값 5.5)을 "준비 중" 자리표시로 안내한다. 관리자 전용 데이터(예: 전체 직원 수·환자 수 요약)를 **관리자 보호 API**로 불러와 표시한다.
6. **로그인한 직원만**(NFR3) 모든 동작에 접근한다. 기존 로그인·환자·안전·흐름 기능은 **전혀 깨지지 않는다**(일반 직원은 지금까지처럼 모든 환자 기능 사용 — 권한 세분화는 5.3).
7. 관리자 보호 API(`/api/admin/*`)는 일반 직원 호출 시 **403**, 미인증 시 **401**을 반환한다.

> **범위 밖(하지 말 것):**
> - **직원 계정 등록/수정/삭제·직군 지정 = Story 5.2(FR11)** — 이번엔 관리자 화면 "껍데기"(진입+가드+랜딩)만. CRUD 없음.
> - **직원별 접근 권한(정보 범위) 설정 = Story 5.3(FR12)** — 이번엔 admin vs staff 2단계만. 세분화 권한·일반 직원의 화면 범위 제한 없음.
> - **전체 현황 대시보드(오늘 환자 수·과별 혼잡도·평균 대기시간 그래프) = Story 5.4(FR13)** — 이번 `/admin` 랜딩의 요약 숫자는 최소 자리표시(전체 직원·환자 수 정도)이지 FR13 대시보드가 아니다.
> - **기준값 설정(대기 초과 시간 등) = Story 5.5(FR14)** — `STAGE_OVERDUE_MINUTES`는 계속 하드코딩.
> - **접근 감사 로그**(누가 무엇을 봤는지 기록) — 후속(1.4/2.1 deferred). 이번 범위 아님.
> - **역할 변경 UI**(관리자가 다른 직원을 admin으로 승격) = 5.2 직원 관리에서.

## Dev Notes

### 이 스토리의 핵심 (한 줄)
**첫 RBAC(역할 기반 접근제어) 도입.** `Staff.role`에 `admin`을 두고, 백엔드 `get_current_admin`(403 게이트) + 관리자 전용 `/api/admin/*` + 프론트 `AdminGuard`·`/admin` 랜딩·조건부 "관리자" 메뉴. **껍데기만**(실제 관리 기능은 5.2~5.5).

### ⚠️ 이 스토리의 가장 중요한 설계 판단

1. **진짜 자물쇠는 백엔드 `get_current_admin`.** 프론트의 메뉴 숨김(AppShell)·화면 차단(AdminGuard)은 UX 편의일 뿐, 보안은 백엔드가 강제한다(403). 일반 직원이 localStorage를 조작하거나 `/admin`에 직접 들어가도 `/api/admin/*`가 403으로 막는다(3.1 "백엔드가 강제" 철학과 동일).

2. **`Staff.role`은 이미 존재** — 모델 변경 없음. 현재 전원 `"staff"`(1.3에서 기본값). 관리자 계정만 `"admin"`. **2단계만**(admin/staff). 세분화 권한(과별·정보범위)은 5.3.

3. **관리자 시드는 멱등(별도 블록).** 기존 Staff 시드는 "테이블이 비었을 때만 nurse1"이라, 이미 nurse1이 있는 DB에선 안 돈다. 그래서 `admin1`은 **username 존재 확인 후 없으면 생성**하는 별도 idempotent 블록으로 추가(4.4 StageEntry 백필·2.x 시드 멱등 패턴). 재시작해도 중복 안 생김.

4. **role 출처는 `/api/auth/me`** — 이미 `role`을 반환(변경 0). AppShell(메뉴)·AdminGuard(게이트)가 각각 `/me`를 호출해 role을 읽는다. localStorage에 role을 저장하지 않음(서버가 진실 출처 → 스푸핑/스테일 위험 회피). 관리자 페이지에서 /me가 몇 번 호출되는 중복은 허용(내부 앱, 무해 — 공유 캐시는 후속 최적화).

5. **`/admin` 랜딩은 자리표시.** 5.2~5.5 기능 카드는 "준비 중" 비활성. 요약 숫자(직원·환자 수)는 게이트 증명 + 최소 콘텐츠일 뿐, FR13 대시보드(5.4) 아님. UX는 기존 디자인 토큰·AppShell·QuickCard 톤 재사용(관리자 화면 UX는 별도 설계 없음 — 구현준비 점검 Major #3에서 보강 권고됐으나 1차는 단순 일관 화면).

6. **RSC 프리렌더 금지(1.4).** 관리 데이터도 Server Component에서 프리렌더하지 않고 AdminGuard 통과 후 클라이언트에서 토큰 달아 호출.

# security.py 에 추가 (import은 이미 존재: Depends, HTTPException, status, Staff)
def get_current_admin(current: Staff = Depends(get_current_user)) -> Staff:
    """관리자 전용 보호. 로그인했어도 admin 아니면 403."""
    if current.role != "admin":
        raise HTTPException(
            status_code=status.HTTP_403_FORBIDDEN, detail="관리자 권한이 필요합니다"
        )
    return current

```tsx
// AdminGuard.tsx (NEW, 골격) — AuthGuard 본떠 role 체크 추가
"use client";
import { useEffect, useState, type ReactNode } from "react";
import { useRouter } from "next/navigation";
import { API_BASE } from "@/lib/api";
import { authHeader, clearToken, getToken } from "@/lib/auth";

const ME_TIMEOUT_MS = 8000;
export function AdminGuard({ children }: { children: ReactNode }) {
  const router = useRouter();
  const [status, setStatus] = useState<"checking" | "authed">("checking");
  useEffect(() => {
    const token = getToken();
    if (!token) { router.replace("/login"); return; }
    let cancelled = false;
    const controller = new AbortController();
    const timeout = setTimeout(() => controller.abort(), ME_TIMEOUT_MS);
    (async () => {
      try {
        const res = await fetch(`${API_BASE}/api/auth/me`, { headers: { ...authHeader() }, signal: controller.signal });
        if (cancelled) return;
        if (res.status === 401 || res.status === 403) { clearToken(); router.replace("/login"); return; }
        if (!res.ok) { router.replace("/login"); return; }
        const data = await res.json();
        if (cancelled) return;
        if (data?.role !== "admin") { router.replace("/"); return; } // 일반 직원 차단
        setStatus("authed");
      } catch { if (!cancelled) router.replace("/login"); }
      finally { clearTimeout(timeout); }
    })();
    return () => { cancelled = true; controller.abort(); clearTimeout(timeout); };
  }, [router]);
  if (status === "checking") {
    return (<main className="flex min-h-screen items-center justify-center bg-bg-base p-8">
      <p className="text-text-secondary" role="status">확인 중…</p></main>);
  }
  return <>{children}</>;
}
```
