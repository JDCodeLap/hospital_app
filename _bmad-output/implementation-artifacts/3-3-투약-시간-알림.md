## Story

As a 병원 직원,
I want 환자에게 정해진 투약 시간이 되면 "투약 시간입니다" 알림을 모아 보고, 투약을 완료 체크하면 그 알림이 정리되기를,
so that 제때 약을 주고 빠뜨리지 않는다. (FR5)

## Acceptance Criteria

epics.md 원문(Story 3.3) + EXPERIENCE.md(Flow 3 "시간 됨→🔔→완료 체크→초록·정리", 알림 모아보기 화면, 투약시간=앰버) + 3.1(Medication.schedule)·2.4(broadcast)·1.4(AuthGuard)에서 도출한 검증 가능한 항목입니다.

1. **(원문)** 환자에게 투약 시간(`Medication.schedule`, 예 "08:00,20:00")이 설정된 상태에서, **그 시간이 지나면(현재 시각 ≥ 예정 시각, 오늘 분)** 그 투약이 "받을 시간이 된 알림"으로 표시된다.
2. **알림 모아보기 화면**(`/alerts`, AuthGuard 보호)에서 현재 시간이 된(아직 완료 안 한) 투약 알림 목록을 본다 — 각 항목: **환자명 · 약 이름(용량) · 예정 시각**, 앰버(`warning`) 톤. 비어 있으면 "표시할 항목이 없습니다", 로딩/오류 상태 처리.
3. **(원문)** 알림의 **"완료" 버튼**을 누르면 그 투약(해당 슬롯)이 **완료로 기록**되고 목록에서 **정리(사라짐)** 된다. → 같은 시간 슬롯은 그날 다시 알림에 안 뜬다.
4. 백엔드에 **완료 기록 테이블** `MedicationAdministration`(투약 1건 = `medication_id` + `scheduled_time`(HH:MM) + `administered_date`(날짜) + `administered_by` + `administered_at`, **(medication_id, scheduled_time, administered_date) unique** = 같은 약·같은 시각·같은 날 1건)를 추가한다. NEW 테이블 → `create_all`로 생성(마이그레이션 불필요).
5. 백엔드 엔드포인트 2종(둘 다 `get_current_user` 보호):
   - `GET /api/medication-alerts` — **현재 시각 기준 due(시간 지남 + 미완료)** 인 active 투약을 전 환자에서 모아 반환(환자명 포함). schedule의 비시간 토큰("필요시" 등)은 무시.
   - `POST /api/medications/{medication_id}/administer`(body `{scheduled_time}`) — 그 슬롯을 오늘 완료로 기록. 약 없으면 404, scheduled_time이 그 약 schedule에 없으면 400, 이미 기록되어 있으면 멱등(중복 생성 X, IntegrityError 방어). 저장 후 `manager.broadcast(patient_id, {"type":"patient_updated",...})`(2.4 재사용).
6. 프론트 알림 화면: 항목의 **"완료"** → `POST .../administer` → 성공 시 목록에서 제거(또는 reload). 항목은 환자 통합 화면(`/patients/{id}`)으로 가는 링크도 제공(Flow 3 "알림 누르면 투약 정보로 이동"). 401/403 → `clearToken()`+`/login`, 기타 오류 → 인라인. 완료 버튼 재진입 가드(3.1/3.2 교훈). 홈에 **"알림 모아보기" 진입 링크** 추가.
7. **시간이 흐르면 새 알림이 생기는** 특성상, 알림 화면은 **열 때 서버 시각 기준으로 계산**하고 **새로고침 버튼 + 가벼운 자동 갱신(예 60초)** 으로 최신화한다(시간 기반이라 정당 — 데이터 변경 폴링 아님). 기존 2.x/3.x 화면 동작(AuthGuard·홈·통합화면)을 깨지 않는다.

> **범위 밖(하지 말 것):** 진짜 예약 푸시/스케줄러(cron으로 정확히 그 시각에 OS/모바일 푸시) = 후속/배포(현재는 화면 열 때·새로고침 시 서버 계산 = pull). / 홈 상단 🔔 안 읽은 개수 배지 = 폴리시(후속). / **담당 직원 배정 기반 필터**(내 환자만) = 현재 전 직원이 모든 알림 봄(NFR2 단순 권한), 역할·배정은 Epic 5. / 요일/간격(매주·격일) 등 복잡 스케줄 = HH:MM 일일 반복만. / 대기 초과 알림=4.4 / 다른 알림 종류 통합(검사·안전 등 한 화면 집계)=후속. / 투약 완료가 `Medication.status`를 바꾸지 않음(슬롯별 기록만 — 반복 투약 유지).

## Dev Notes

### 이 스토리의 핵심 (한 줄)
투약 시간(`Medication.schedule`)이 지난 미완료 투약을 **서버가 현재 시각 기준으로 계산**해 "알림 모아보기" 화면에 모으고, **"완료"를 누르면 그 슬롯을 완료로 기록**(`MedicationAdministration`)해 목록에서 정리한다. 완료는 (약·시각·날짜) 단위라 반복 투약을 깨지 않는다.

### ⚠️ 이 스토리의 가장 중요한 설계 판단
- **"시간이 되면"은 서버 시각 기준 pull 계산.** 진짜 예약 푸시(cron으로 정확히 그 시각에 알림 발사)는 데모 범위 밖. 대신 알림 화면을 **열 때/새로고침/60초마다** 서버가 `현재시각 ≥ 예정시각 AND 미완료`인 투약을 계산해 보여준다. → AC("시간이 되면 표시")를 단순·정직하게 만족. (시간 기반 자동 갱신은 2.4의 '데이터 변경 폴링 금지'와 무관 — 시간 흐름 자체가 트리거)
- **완료는 슬롯 단위로 기록한다(상태 토글 아님).** `schedule="08:00,20:00"`은 매일 반복이므로 `Medication.status`를 done으로 바꾸면 안 된다. `MedicationAdministration`에 **(medication_id, scheduled_time, administered_date) 1건**을 남겨, "오늘 08:00분은 줬다"를 표현 → 그 슬롯만 알림에서 빠지고 20:00·내일은 그대로 뜬다.
- **새 테이블로 마이그레이션 회피**(3.2와 동일 정책). 멱등은 **unique 제약 + IntegrityError 방어**(3.2 교훈 그대로).
- **"담당 직원"은 현재 전 직원.** 환자-직원 배정 개념이 아직 없음(NFR2 단순 권한) → 모든 로그인 직원이 모든 due 알림을 본다. 역할·배정 기반 필터는 Epic 5. (스토리 원문의 "담당 직원에게"를 이 범위에선 "로그인 직원에게"로 해석)

- **시간 파싱(요지) — `main.py`:**
  ```python
  import re
  _HM = re.compile(r"^([01]\d|2[0-3]):[0-5]\d$")
  def parse_schedule_times(schedule: str) -> list[str]:
      out = []
      for tok in (schedule or "").split(","):
          t = tok.strip()
          if _HM.match(t) and t not in out:
              out.append(t)
      return sorted(out)
  ```
- **알림 목록(요지):**
  ```python
  @app.get("/api/medication-alerts")
  def medication_alerts(current: Staff = Depends(get_current_user),
                        session: Session = Depends(get_session)) -> list[dict]:
      now = datetime.now(); today = now.date(); cur = now.strftime("%H:%M")
      meds = session.exec(select(Medication).where(Medication.status == "active")).all()
      alerts = []
      for med in meds:
          for t in parse_schedule_times(med.schedule):
              if t <= cur:  # 시간이 지남(오늘 분)
                  done = session.exec(select(MedicationAdministration).where(
                      MedicationAdministration.medication_id == med.id,
                      MedicationAdministration.scheduled_time == t,
                      MedicationAdministration.administered_date == today)).first()
                  if done is None:
                      p = session.get(Patient, med.patient_id)
                      alerts.append({"medication_id": med.id, "patient_id": med.patient_id,
                                     "patient_name": p.name if p else "", "drug_name": med.drug_name,
                                     "dose": med.dose, "scheduled_time": t})
      alerts.sort(key=lambda a: (a["scheduled_time"], a["patient_name"]))
      return alerts
  ```
- **완료 기록(요지):**
  ```python
  class AdministerIn(BaseModel):
      scheduled_time: str
  @app.post("/api/medications/{medication_id}/administer")
  async def administer_medication(medication_id: int, payload: AdministerIn,
        current: Staff = Depends(get_current_user), session: Session = Depends(get_session)) -> dict:
      med = session.get(Medication, medication_id)
      if med is None: raise HTTPException(404, "투약 정보를 찾을 수 없습니다")
      if payload.scheduled_time not in parse_schedule_times(med.schedule):
          raise HTTPException(400, "유효한 투약 시간이 아닙니다")
      rec = MedicationAdministration(medication_id=medication_id, scheduled_time=payload.scheduled_time,
            administered_date=datetime.now().date(), administered_by=current.id, administered_at=datetime.now())
      session.add(rec)
      try:
          session.commit()
      except IntegrityError:  # 이미 완료(같은 슬롯) → 멱등
          session.rollback()
      else:
          await manager.broadcast(med.patient_id, {"type": "patient_updated", "patient_id": med.patient_id})
      return {"medication_id": medication_id, "scheduled_time": payload.scheduled_time, "status": "administered"}
  ```
- **프론트 알림 화면(요지):** `PatientSearch` 패턴 복제 — 로드(AbortController) + 60초 `setInterval` 갱신(cleanup에서 clear) + 새로고침 버튼. 항목별 완료는 `submittingRef`(또는 처리중 id 집합)로 가드. 성공 시 그 항목 제거 또는 전체 reload. 앰버 카드: `bg-warning/15 text-warning`, 🔔/⏰ 아이콘 + "예정 {time}". 빈 상태 문구 "표시할 항목이 없습니다".

- ❌ 자동 갱신 `setInterval` cleanup 누락 → 화면 떠난 뒤에도 요청. effect cleanup에서 clear.
- ❌ 완료 버튼 항목별 가드 없이 전역 submitting → 한 항목 처리 중 다른 항목 막힘. 항목별(처리중 id) 또는 ref로.
