# Story 2.3: 환자 통합 화면

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 병원 직원,
I want 환자 한 명을 선택하면 그 사람의 모든 정보를 한 화면에서 보기를,
so that 여러 프로그램을 열지 않아도 된다. (FR1)

## Acceptance Criteria

epics.md 원문 + EXPERIENCE(환자 통합 화면)·UX 목업(patient-screen.html)·2.1(데이터/조회 API)·2.2(검색 카드 진입)·1.4(보호 패턴)에서 도출한 검증 가능한 항목입니다.

1. **(원문)** 검색 결과에서 환자 한 명을 선택하면(=카드 클릭 → `/patients/{id}`), 그 환자의 **기본정보·방문기록·진단·투약·검사결과·수납**이 한 화면에 **카드형**으로 표시된다.
2. **(원문)** **UX 목업과 같은 다크 모드 레이아웃**을 따른다 — 상단바(뒤로가기+제목) → (알레르기 있으면) 빨간 안전 경고 배너 → 환자 헤더(이름·나이·성별·등록번호·현재단계) → 정보 카드들. (patient-screen.html / DESIGN.md)
3. 화면(`/patients/[id]`)은 **로그인한 직원만** 접근 — `AuthGuard`로 감싸고, 환자 데이터는 **통과(authed) 후 토큰 달아 클라이언트에서 호출**한다(미로그인 차단 + RSC 프리렌더 금지 — 1.4 리뷰 가이드).
4. 데이터는 **기존 `GET /api/patients/{id}`(2.1, 보호됨)** 를 그대로 사용한다(새 엔드포인트·백엔드 변경 없음). 응답 묶음: `patient`(기본정보+allergies) + `visits`·`diagnoses`·`medications`·`lab_results`·`billings`.
5. **알레르기가 있으면** 화면 상단(헤더 위)에 **빨간 안전 경고 배너**("⚠ {알레르기} — 처방 시 주의")를 표시한다(색만 금지: 아이콘+글자). **단, 배너의 "확인 전까지 유지/닫기" 상호작용은 범위 밖(=Story 3.2).** 여기선 정적 표시만.
6. 각 정보 카드 내용:
   - **방문기록**: 방문일시·부서·사유
   - **진단**: 진단명·진단일·(상태)
   - **투약**: 약 이름·용량·투약시간(schedule 텍스트 그대로). *시간 도래 알림/완료체크는 범위 밖(3.3)*
   - **검사결과**: 검사명·값+단위·이상여부(`flag`=abnormal이면 주의 배지, 색+글자)
   - **수납**: 항목·금액(원 표기)·상태(paid=완료 초록 / unpaid=미납 배지)
7. **로딩/없음/오류 상태**를 구분해 보여준다 — 불러오는 중(스켈레톤 또는 "불러오는 중…"), 없는 환자(`404` → "환자를 찾을 수 없습니다"), 백엔드 연결 실패(친절 안내). 세션 만료(`401/403`)면 `clearToken()` 후 `/login`으로(2.2와 동일 정책).
8. 상단바의 **뒤로가기**(또는 "← 목록/홈")로 검색 화면이나 홈으로 돌아갈 수 있다.
9. 다크 토큰·접근성(터치 44px, 색만으로 정보전달 금지)을 따른다. 빈 하위목록(예: 검사 없음)은 카드 안에서 "기록 없음"으로 안내한다.

> **범위 밖(하지 말 것):** 단계 타임라인 그래픽(접수→진료→검사→수납 진행바) = **Story 4.1** (여기선 현재단계를 텍스트/배지로만 표시). 필수 절차 체크리스트 = **3.4**. 투약 시간 알림/완료체크 = **3.3**. 약물 충돌 경고 팝업 = **3.1**. 안전배너 확인-유지/닫기 = **3.2**. 실시간 자동 갱신(WebSocket) = **2.4**. 알림 벨/하단 탭바 등 전역 네비게이션 = 후속.

## Dev Notes

### 이 스토리의 핵심 (한 줄)
검색 카드를 누르면 가는 **`/patients/[id]` 상세 화면**을 만들어, 이미 있는 `GET /api/patients/{id}`(2.1) 데이터를 **목업과 같은 다크 카드 레이아웃**으로 한 화면에 모아 보여준다. (읽기 전용 · 안전배너는 정적 표시)

### 2.1에서 이어받는 사실 — 반드시 인지(새로 만들지 말 것)
- ✅ **백엔드 `GET /api/patients/{id}`(보호됨)가 이미 완성**되어 있음 → **백엔드 변경 없음.** 없는 id면 404. [Source: backend/app/main.py(get_patient)]
- ✅ 응답 묶음 구조(중요):
  ```json
  {
    "patient": { "id", "registration_number", "name", "age", "birth_date"(ISO), "gender"("M"/"F"/"기타"), "allergies"(콤마구분 문자열, 없으면 ""), "current_stage"("접수"/"진료"/"검사"/"수납") },
    "visits":      [ { "visited_at"(ISO), "department", "reason", ... } ],
    "diagnoses":   [ { "name", "diagnosed_at"(ISO date), "status"("active"/"resolved"), ... } ],
    "medications": [ { "drug_name", "dose", "schedule"(예 "08:00,20:00"), "status"("active"/"done"), ... } ],
    "lab_results": [ { "test_name", "value", "unit", "flag"("normal"/"abnormal"), "measured_at"(ISO), ... } ],
    "billings":    [ { "item", "amount"(정수, 원), "status"("paid"/"unpaid"), ... } ]
  }
  ```
  - 각 자식은 id 순 정렬되어 옴(순서 안정). 자식 객체엔 `id`/`patient_id`도 포함되나 화면에선 위 필드만 사용.
- ✅ 샘플 환자 8명 시드: 김철수(P0001, 페니실린 알레르기·진단2·투약2·검사2·수납1·방문1), 그 외 다양. 검증에 사용. [Source: backend/app/main.py(seed_patients)]

### 2.2에서 이어받는 프론트 자산 — 재사용 (패턴 그대로)
- ✅ **`AuthGuard`**(`components/AuthGuard.tsx`): 상세 화면을 이걸로 감쌈(미로그인/무효토큰 차단).
- ✅ **`lib/api.ts`(`API_BASE`) + `lib/auth.ts`(`authHeader()`, `clearToken()`)**: 보호 요청 + 401 처리에 사용.
- ✅ **2.2 `PatientSearch.tsx`의 fetch 패턴**을 그대로 따른다: `AbortController` + `cancelled` 표식 + 401/403→`clearToken()`+`router.replace("/login")` + 404 구분 + 오류 안내. **상세는 검색어가 없으니 디바운스는 불필요**(id 바뀔 때 1회 fetch).
- ✅ 카드/배지는 div + 토큰 클래스로 직접(별도 UI 부품 불필요). 2.2 카드 스타일과 톤 맞춤.
- ✅ 검색 카드 링크는 이미 `/patients/{id}`로 연결되어 있음(2.2). 이 스토리가 그 대상 페이지를 만든다 → 클릭 시 더 이상 404 아님. (2.2에서 건 `prefetch={false}`는 이제 페이지가 생겼으니 두어도/풀어도 무방 — 굳이 손대지 말 것)

### ⚠️ 1.4 코드리뷰 가이드 — 이 스토리에서 꼭 지킬 것 (PHI)

  ```tsx
  "use client";
  import { useEffect, useState } from "react";
  import { useParams, useRouter } from "next/navigation";
  import { API_BASE } from "@/lib/api";
  import { authHeader, clearToken } from "@/lib/auth";

  type PatientBundle = {
    patient: { id:number; registration_number:string; name:string; age:number;
      birth_date:string; gender:string; allergies:string; current_stage:string };
    visits: { visited_at:string; department:string; reason:string }[];

    medications: { drug_name:string; dose:string; schedule:string; status:string }[];
    lab_results: { test_name:string; value:string; unit:string; flag:string; measured_at:string }[];
    billings: { item:string; amount:number; status:string }[];
  };

  const { id } = useParams<{ id: string }>();
  // useEffect: cancelled 표식 + AbortController, res.status===404 → notFound 상태,

  ```
