# Story 3.2: 안전 경고 배너 유지

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 병원 직원,
I want 환자에게 위험 정보(알레르기 등)가 있으면 통합 화면 맨 위에 빨간 경고 배너가 계속 보이고, 담당자가 "확인"하기 전까지 사라지지 않기를,
so that 누구도 그 위험을 놓치지 않는다. (FR7)

## Acceptance Criteria

epics.md 원문(Story 3.2) + PRD FR7("안전 알림은 확인·조치 전까지 사라지지 않는다") + 2.3(정적 배너)·2.4(실시간 broadcast)·3.1(safety/처방 파이프)에서 도출한 검증 가능한 항목입니다.

1. **(원문)** 환자에게 위험 정보가 있을 때 통합 화면(`/patients/[id]`)을 열면 **빨간 경고 배너가 화면 최상단**(환자 헤더보다 위)에 표시된다. (현재 "위험 정보" = `allergies` 비어있지 않음)
2. **(원문)** 그 경고 배너는 **담당자가 "확인" 버튼을 누르기 전까지 사라지지 않는다** — 화면을 닫았다 다시 열거나, 다른 직원이 같은 환자를 열어도 **미확인 상태면 계속 빨간 경고로 보인다**(브라우저 새로고침/재진입에도 유지).
3. **확인은 백엔드에 기록**된다(프론트 로컬 상태만 아님 — "누구도 놓치지 않는다" 보장). 새 테이블 `SafetyAck`(환자당 1건: `patient_id` unique, `acknowledged_by`(직원 id), `acknowledged_at`)에 저장. → 누가/언제 확인했는지 남는다.
4. 백엔드에 **확인 기록 엔드포인트** `POST /api/patients/{patient_id}/safety-ack`(보호됨, `get_current_user`)를 추가한다. 환자 없으면 404. 이미 확인 기록이 있으면 그대로 두고(멱등) 현재 상태 반환, 없으면 현재 직원·시각으로 생성. 저장 후 **`manager.broadcast(patient_id, {"type":"patient_updated", ...})`**(2.4 파이프 재사용) → 같은 환자를 보는 다른 화면도 새로고침 없이 갱신.
5. `GET /api/patients/{patient_id}`(2.1) 응답 묶음에 **`safety_ack` 필드**를 추가한다 — 미확인이면 `null`, 확인됐으면 `{acknowledged_by, acknowledged_by_name, acknowledged_at}`. (기존 키는 그대로 유지 = 하위호환, 3.1까지의 화면 안 깨짐)
6. 프론트 통합 화면의 배너를 **상태 기반**으로 바꾼다(`SafetyBanner`):
   - **위험 있음 + 미확인** → **빨간 경고 배너**(role="alert", ⚠ + 알레르기 + "처방 시 주의") + **"확인" 버튼**. 버튼 클릭 → `POST .../safety-ack` → 성공 시 조용한 reload(`onSaved`). (확인 처리 중 버튼 비활성, 실패는 인라인 오류·화면 유지)
   - **위험 있음 + 확인됨** → **차분한 확인 배너**(2.3의 정적 스타일 재사용: 빨강 톤이지만 버튼 없음, 알레르기는 영구 위험이라 계속 표시 + "확인: {이름} · {시각}"). 더는 행동을 요구하지 않음.
   - **위험 없음** → 배너 없음(현재와 동일).
7. **3.1/2.3/2.4 동작을 깨지 않는다** — 처방 폼·경고 팝업(3.1), 카드 레이아웃·로딩/404/401/오류(2.3), WebSocket 구독·조용한 갱신(2.4) 모두 유지. 배너만 정적 → 상태 기반으로 교체.

> **범위 밖(하지 말 것):** 알레르기 외의 다른 "위험 정보" 종류 추가(금기 약물 상시 경고·낙상 위험 등 = 후속/확장) — 이번엔 **allergies 기준** 1종. / 확인 해제(되돌리기)·확인 만료·재확인 요구 로직 = 범위 밖(한 번 확인하면 확인 상태 유지). / 투약 시간 알림=3.3 / 필수 절차 체크리스트=3.4 / 역할별 "확인 권한" 제한·확인 감사 리포트 화면 = Epic 5. / 3.1의 처방 오버라이드 감사로그(별개 사안, deferred 유지).

## Dev Notes

### 이 스토리의 핵심 (한 줄)
2.3의 **정적 알레르기 배너**를 "**확인 전까지 유지되는 안전 알림**"으로 승격한다: 위험 정보가 있으면 빨간 배너가 최상단에 뜨고, **담당자가 "확인"을 누르면 그 사실이 백엔드(`SafetyAck`)에 기록**되어, 이후엔 차분한 확인 배너로 바뀐다. 다른 직원·재진입에도 미확인이면 계속 빨갛게 보인다(2.4 broadcast로 실시간 동기화).

### ⚠️ 이 스토리의 가장 중요한 설계 판단
- **확인 상태는 백엔드에 저장한다(프론트 localStorage 아님).** "누구도 놓치지 않는다"(FR7)를 지키려면 확인이 **모든 직원/모든 기기에 공유**돼야 한다. 그래서 `SafetyAck` 테이블에 기록하고, `GET`이 그 상태를 함께 내려준다. (브라우저 로컬 저장이면 다른 직원 화면엔 여전히 미확인이어야 하는데 공유가 안 됨 → 부적합)
- **새 테이블로 추가(기존 테이블 변경 X) → 마이그레이션 회피.** `Patient`에 컬럼을 더하면 `create_all`이 기존 테이블을 ALTER 못 해(1-2 deferred) Alembic이 필요해진다. `SafetyAck`를 **새 테이블**로 만들면 `create_all`로 충분 — 2.1에서 정한 정책과 동일.
- **확인은 멱등(idempotent).** 환자당 `SafetyAck` 1건(unique patient_id). 두 번 눌러도/두 직원이 동시에 눌러도 중복 생성 없이 같은 확인 상태. (3.1 코드리뷰의 "쓰기 멱등성" 교훈 반영 — 여기선 unique 제약으로 보장)
- **확인 후에도 알레르기 정보는 계속 보인다(영구 위험).** AC는 "확인 전까지 사라지지 않는다"지만, 알레르기는 사라지는 정보가 아니므로 확인 후엔 **빨간 경고(행동 요구) → 차분한 확인 배너**로 톤만 낮춘다(완전 제거 아님). 안전상 더 바람직하고 2.3 정적 배너 스타일을 재사용.
- **실시간 동기화는 2.4 파이프 재사용.** 확인 저장 후 `manager.broadcast` → 같은 환자를 보던 다른 화면이 조용히 reload되어 즉시 "확인됨"으로 바뀐다(새 파이프 X).

- **확인 엔드포인트(요지) — `main.py`:**
  ```python
  @app.post("/api/patients/{patient_id}/safety-ack")
  async def acknowledge_safety(
      patient_id: int,
      current: Staff = Depends(get_current_user),
      session: Session = Depends(get_session),
  ) -> dict[str, object]:
      if session.get(Patient, patient_id) is None:
          raise HTTPException(status_code=404, detail="환자를 찾을 수 없습니다")
      ack = session.exec(select(SafetyAck).where(SafetyAck.patient_id == patient_id)).first()
      if ack is None:  # 멱등: 이미 있으면 그대로 둠
          ack = SafetyAck(patient_id=patient_id, acknowledged_by=current.id,
                          acknowledged_at=datetime.now())
          session.add(ack); session.commit(); session.refresh(ack)
          await manager.broadcast(patient_id, {"type": "patient_updated", "patient_id": patient_id})
      return {"patient_id": patient_id, "acknowledged_by": ack.acknowledged_by,
              "acknowledged_at": ack.acknowledged_at.isoformat()}
  ```
