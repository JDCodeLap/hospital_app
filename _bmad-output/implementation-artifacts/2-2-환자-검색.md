# Story 2.2: 환자 검색

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 병원 직원,
I want 이름이나 등록번호로 환자를 검색하기를,
so that 원하는 환자를 빠르게 찾는다. (FR2)

## Acceptance Criteria

epics.md 원문 + EXPERIENCE(환자 검색 화면)·2.1(데이터/조회 API)·1.4(보호 패턴)에서 도출한 검증 가능한 항목입니다.

1. **(원문)** 환자들이 저장된 상태에서 검색창에 **이름 또는 등록번호**를 입력하면, **일치하는 환자 목록이 실시간으로** 보인다.
2. **(원문)** 일치하는 환자가 없으면 **"검색 결과 없음"** 이 표시된다.
3. 백엔드 `GET /api/patients`에 **선택 검색어 `q`** 추가 — `name` 또는 `registration_number`에 부분일치(대소문자 무시). `q`가 비면 전체 목록. (2.1의 name·등록번호 인덱스 활용) 기존 응답 형태(요약) 유지.
4. 검색 결과는 **환자 카드**로 표시 — 이름·나이·현재 단계, 그리고 **알레르기가 있으면 위험 배지**(빨강+아이콘+글자, 색만으로 전달 금지). (EXPERIENCE 환자 카드)
5. 화면(`/patients`)은 **로그인한 직원만** 접근 — `AuthGuard`로 감싸고, 환자 데이터는 **통과(authed) 후 토큰 달아 클라이언트에서 호출**한다(미로그인 차단 + RSC 프리렌더 금지 — 1.4 리뷰 가이드).
6. **로딩/빈/오류 상태**를 구분해 보여준다 — 불러오는 중("불러오는 중…"), 결과 없음("검색 결과 없음"), 백엔드 연결 실패(친절 안내). (EXPERIENCE State Patterns)
7. 홈(`/`)에서 검색 화면으로 가는 진입(버튼/링크)이 있고, 검색 화면에서 환자 카드를 누르면 **`/patients/{id}`(통합 화면)로 이동**한다. (통합 화면 자체 구현은 **Story 2.3**)
8. 다크 토큰·접근성(터치 44px=`h-11`, 색만으로 정보전달 금지)을 따른다.

> **범위 밖(하지 말 것):** 환자 통합 화면(상세 카드 레이아웃) 자체 = **Story 2.3**. 실시간 자동 갱신(WebSocket) = **Story 2.4**. 역할별 권한 = Epic 5.

### 핵심 코드 가이드

- **백엔드 검색(q):**
  ```python
  from sqlalchemy import or_, col  # col은 sqlmodel에서도 import 가능: from sqlmodel import col
  @app.get("/api/patients")
  def list_patients(
      q: str | None = None,
      current: Staff = Depends(get_current_user),
      session: Session = Depends(get_session),
  ) -> list[dict[str, object]]:
      stmt = select(Patient)
      if q:
          like = f"%{q}%"
          stmt = stmt.where(or_(col(Patient.name).ilike(like),
                                col(Patient.registration_number).ilike(like)))
      patients = session.exec(stmt.order_by(Patient.registration_number)).all()
      return [{"id": p.id, "registration_number": p.registration_number, "name": p.name,
               "age": calc_age(p.birth_date), "gender": p.gender,
               "current_stage": p.current_stage, "allergies": p.allergies} for p in patients]
  ```
  - `ilike`는 PostgreSQL 대소문자 무시 부분일치. `col(...)`로 타입 경고 없이 `.ilike` 사용.
  - **목록 요약에 `allergies` 추가**(카드 위험 배지용). 기존 키는 유지(하위호환).

- **프론트 디바운스 검색(요지):**
  ```tsx
  "use client";
  import { useEffect, useState } from "react";
  import { API_BASE } from "@/lib/api";
  import { authHeader } from "@/lib/auth";

  type PatientSummary = { id: number; registration_number: string; name: string;
    age: number; gender: string; current_stage: string; allergies: string };

  // query가 바뀌면 250ms 후 fetch, 그 사이 또 바뀌면 이전 요청 취소
  useEffect(() => {
    const controller = new AbortController();
    const t = setTimeout(async () => {
      try {
        const url = `${API_BASE}/api/patients` + (query ? `?q=${encodeURIComponent(query)}` : "");
        const res = await fetch(url, { headers: { ...authHeader() }, signal: controller.signal });
        if (!res.ok) throw new Error();
        setResults(await res.json()); setError(null);
      } catch (e) {
        if (e instanceof DOMException && e.name === "AbortError") return;
        setError("백엔드에 연결할 수 없습니다…");
      } finally { setLoading(false); }
    }, 250);
    return () => { controller.abort(); clearTimeout(t); };
  }, [query]);
  ```

- **카드(요지):** `<Link href={'/patients/' + p.id}>` 안에 `bg-bg-surface` 카드. 이름/나이/단계 + `p.allergies && <span class="...danger...">⚠ 알레르기</span>`.

### 흔한 실수 방지
- ❌ 새 검색 엔드포인트를 따로 만들기 → 기존 `GET /api/patients`에 `q`만 추가.
- ❌ 환자 데이터를 Server Component에서 prerender → 미인증 노출. 반드시 통과 후 클라이언트 fetch.
- ❌ 디바운스만 하고 AbortController 안 씀 → 늦은 응답이 최신 결과 덮어씀.
- ❌ 알레르기를 색(빨강)만으로 표시 → 아이콘+글자 동반(UX-DR8).
- ❌ 통합 화면(상세 레이아웃) 만들기 → Story 2.3 범위.
- ❌ WebSocket 실시간 갱신 → Story 2.4 범위. (여기 "실시간"은 타이핑 즉시 필터를 의미)
- ❌ `q`에 사용자 입력을 문자열 포매팅으로 SQL에 직접 넣기 → SQLAlchemy `ilike` 파라미터 바인딩 사용(인젝션 방지). f-string은 `%{q}%` 패턴 값에만, 쿼리에 바인딩되게.
