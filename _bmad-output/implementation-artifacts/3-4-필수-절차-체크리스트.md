# Story 3.4: 필수 절차 체크리스트

Status: done

## Story

As a 병원 직원,
I want 시술/검사 같은 주요 절차로 넘어가기 전에 필수 항목(본인확인·금식·동의서 등)을 체크리스트로 확인하고, 모두 체크해야만 "다음 단계로" 버튼이 켜지기를,
so that 빠뜨리는 절차 없이 안전하게 다음 단계로 진행한다. (FR6)

2. **(원문)** **모든 항목이 체크된 경우에만** "다음 단계로" 버튼이 **활성화(초록 `success` 톤)** 된다. **하나라도 미체크면 버튼은 비활성(회색)** 이고 눌리지 않는다(`disabled`). (UX-DR6, EXPERIENCE line 70)
3. "다음 단계로" 버튼을 누르면 그 환자의 **현재 단계(`current_stage`)가 다음 단계로 진행**된다 — 단계 순서는 **접수 → 진료 → 검사 → 수납**. 진행 후 **체크 상태는 초기화**(다음 절차를 위해 비움)되고, 통합 화면의 현재 단계 배지가 갱신된다. (FR6 "다 체크해야 다음 단계로 넘어간다")
4. 이미 **마지막 단계(수납)** 인 환자는 다음 단계가 없으므로 "다음 단계로" 버튼 대신 "최종 단계" 같은 안내를 보이고 진행 동작은 하지 않는다(또는 버튼 숨김). 체크리스트 항목 자체는 계속 보여도 된다.

## Dev Notes

### 이 스토리의 핵심 (한 줄)
환자 통합 화면에 **필수 절차 체크리스트**를 띄우고, **모든 항목을 체크해야만 "다음 단계로" 버튼이 켜진다.** 체크 상태는 백엔드(`ChecklistCheck`)에 (환자·항목) 단위로 저장돼 모든 직원에게 공유되고, 진행 버튼을 누르면 `current_stage`가 다음 단계로 올라가며 체크가 초기화된다. 진행은 **서버에서도 미완료면 거부(409)** 한다.

### ⚠️ 이 스토리의 가장 중요한 설계 판단
- **체크 상태는 백엔드에 기록(프론트 로컬 저장 아님).** 3.2 SafetyAck·3.3 MedicationAdministration과 같은 "(대상·항목) 기록 테이블" 패턴을 재사용 → 새로고침/다른 직원/재진입에 체크가 공유·유지(FR6 "빠뜨리는 절차 없이"의 본질). [Source: 3-3 인수인계 노트, backend/app/models.py(SafetyAck, MedicationAdministration)]
- **3.3과의 차이 — 날짜키를 쓰지 않는다.** 투약(MedicationAdministration)은 매일 반복이라 `administered_date`로 슬롯을 날마다 구분했지만, **절차 체크리스트는 "시술 1회 전" 확인**이라 날마다 반복되는 개념이 아니다. 그래서 `ChecklistCheck`의 unique는 **(patient_id, item_key)** 로만 둔다(날짜 없음). 다음 절차를 위한 재사용은 "다음 단계로 진행 시 그 환자 체크를 전부 삭제(초기화)"로 처리한다. (이 차이를 의식적으로 둔 것 — 3.3 패턴을 그대로 베끼되 날짜 차원만 뺀다)
- **안전 강제는 백엔드 409.** 프론트 버튼 비활성은 UX일 뿐, 진짜 자물쇠는 `advance-stage`가 "모든 항목 체크" 아니면 409로 거부(3.1의 "확인 없이는 저장 거부 409"와 같은 철학). API 직접 호출로도 절차를 건너뛸 수 없게 한다.
- **"다음 단계로"는 current_stage를 실제로 올린다(FR6 충족).** 단순히 버튼만 켜고 끝내면 FR6의 "다음 단계로 넘어간다"가 빈 동작이 된다. 그래서 진행 시 `STAGE_ORDER`(접수→진료→검사→수납)에서 다음 값으로 `current_stage`를 변경한다. **단, 단계 진행 타임라인 그래픽(점+선 시각화)은 4.1 범위** — 현재 단계는 기존처럼 텍스트 배지로만 표시(2.3 그대로). 마지막 단계(수납)면 진행 불가(400/버튼 숨김).
- **필수 항목은 백엔드 상수(단일 표준 세트).** 단계별 다른 항목·관리자 편집은 범위 밖. `CHECKLIST_ITEMS`(본인확인·금식·동의서)를 백엔드가 정의하고, 프론트는 받아서 그린다(라벨도 백엔드 제공 → 한 곳에서 관리).

  class ChecklistCheck(SQLModel, table=True):

      __table_args__ = (UniqueConstraint("patient_id", "item_key"),)
      id: int | None = Field(default=None, primary_key=True)
      patient_id: int = Field(foreign_key="patient.id", index=True)
      item_key: str            # "identity" / "fasting" / "consent"

- **상수·헬퍼(요지) — `main.py`:**
  ```python
  CHECKLIST_ITEMS = [
      {"key": "identity", "label": "본인 확인"},
      {"key": "fasting", "label": "금식 확인"},
      {"key": "consent", "label": "동의서 확인"},
  ]
  CHECKLIST_KEYS = {it["key"] for it in CHECKLIST_ITEMS}
  STAGE_ORDER = ["접수", "진료", "검사", "수납"]

  def next_stage(stage: str) -> str | None:
      if stage in STAGE_ORDER:
          i = STAGE_ORDER.index(stage)
          if i + 1 < len(STAGE_ORDER):
              return STAGE_ORDER[i + 1]
      return None
  ```

- **체크 토글(요지):**
  ```python
  class ChecklistToggleIn(BaseModel):
      checked: bool

  @app.post("/api/patients/{patient_id}/checklist/{item_key}")
  async def toggle_checklist(patient_id: int, item_key: str, payload: ChecklistToggleIn,
        current: Staff = Depends(get_current_user), session: Session = Depends(get_session)) -> dict:
      if session.get(Patient, patient_id) is None:
          raise HTTPException(404, "환자를 찾을 수 없습니다")
      if item_key not in CHECKLIST_KEYS:
          raise HTTPException(400, "유효한 체크 항목이 아닙니다")
      row = session.exec(select(ChecklistCheck).where(
          ChecklistCheck.patient_id == patient_id,
          ChecklistCheck.item_key == item_key)).first()
      if payload.checked and row is None:
          session.add(ChecklistCheck(patient_id=patient_id, item_key=item_key,
                      checked_by=current.id, checked_at=datetime.now()))
          try:
              session.commit()
          except IntegrityError:        # 동시 체크 → 멱등
              session.rollback()
          else:
              await manager.broadcast(patient_id, {"type": "patient_updated", "patient_id": patient_id})
      elif not payload.checked and row is not None:
          session.delete(row); session.commit()
          await manager.broadcast(patient_id, {"type": "patient_updated", "patient_id": patient_id})
      return {"patient_id": patient_id, "item_key": item_key, "checked": payload.checked}
  ```
- **단계 진행(요지) — 안전 강제 + 초기화:**
  ```python
  @app.post("/api/patients/{patient_id}/advance-stage")
  async def advance_stage(patient_id: int,
        current: Staff = Depends(get_current_user), session: Session = Depends(get_session)) -> dict:
      patient = session.get(Patient, patient_id)
      if patient is None:
          raise HTTPException(404, "환자를 찾을 수 없습니다")
      nxt = next_stage(patient.current_stage)
      if nxt is None:
          raise HTTPException(400, "마지막 단계입니다")
      checked = {c.item_key for c in session.exec(select(ChecklistCheck).where(
          ChecklistCheck.patient_id == patient_id)).all()}
      if not CHECKLIST_KEYS.issubset(checked):       # 미완료면 거부(서버 강제)
          raise HTTPException(409, "필수 절차가 모두 완료되지 않았습니다")
      patient.current_stage = nxt
      for c in session.exec(select(ChecklistCheck).where(
              ChecklistCheck.patient_id == patient_id)).all():
          session.delete(c)                          # 다음 절차 위해 초기화
      session.add(patient); session.commit()
      await manager.broadcast(patient_id, {"type": "patient_updated", "patient_id": patient_id})
      return {"patient_id": patient_id, "current_stage": nxt}
  ```
