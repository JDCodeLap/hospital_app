# Story 2.1: 환자 데이터 구조 + 샘플 데이터

Status: done

## Story

As a 개발자(YC),
I want 환자 정보를 담는 데이터 구조(테이블)와 연습용 샘플 환자 몇 명을,
so that 다음 스토리(검색·통합 화면)를 실제 데이터로 만들 수 있다.

## Acceptance Criteria

epics.md 원문 + architecture(FastAPI+PostgreSQL)·EXPERIENCE(환자 통합 화면 항목)·1.2~1.4 패턴에서 도출한 검증 가능한 항목입니다.

1. **(원문)** PostgreSQL 창고에 환자 정보 구조(이름·나이(생년월일)·등록번호·진단·투약·검사·수납 등)를 만들고 샘플을 넣으면, **샘플 환자 3~5명이 저장되어 조회된다.**
2. **환자(Patient) 테이블**과 관련 정보 테이블(방문기록·진단·투약·검사결과·수납)이 추가되고, 각 관련 정보는 **어느 환자 것인지(patient_id)로 연결**된다.
3. **등록번호(registration_number)는 유니크·인덱스**, 이름(name)도 인덱스(2.2 검색 대비).
4. 시작 시 **샘플 환자 3~5명**이 자동으로 들어가며, 각 환자에 진단·투약·검사·수납 몇 건씩 + EXPERIENCE Flow 1에 맞춰 **"김철수" 환자에 페니실린 알레르기**가 들어 있다(Epic 3 안전경고 대비).
5. 백엔드 엔드포인트 2종:
   - `GET /api/patients` → 환자 목록(요약: id·등록번호·이름·나이·성별 등)
   - `GET /api/patients/{patient_id}` → 그 환자의 **모든 정보 묶음**(기본정보 + 방문·진단·투약·검사·수납 리스트). 없는 id면 **404**.
6. 위 두 엔드포인트는 **로그인한 직원만 접근 가능**(NFR3) — `get_current_user` 의존성으로 보호(토큰 없으면 401). 비밀번호 해시 등 민감/불필요 필드는 응답에 넣지 않는다.
7. 시드는 **이미 환자가 있으면 다시 넣지 않는다**(중복 방지). 등록번호 유니크 충돌로 부팅이 깨지지 않게 한다.

### 핵심 코드 가이드 (개발 시 참고)

- **모델(예시 — Patient + 자식 하나):**
  ```python
  from datetime import date, datetime
  from sqlmodel import Field, SQLModel

  class Patient(SQLModel, table=True):
      __tablename__ = "patient"
      id: int | None = Field(default=None, primary_key=True)
      registration_number: str = Field(unique=True, index=True)  # 등록번호
      name: str = Field(index=True)                              # 이름(2.2 검색 대비)
      birth_date: date
      gender: str = "기타"
      allergies: str = ""          # 콤마구분, Epic 3 안전경고용 (예: "페니실린")
      current_stage: str = "접수"  # 접수/진료/검사/수납 (Epic 4 대비 기본값)

  class Diagnosis(SQLModel, table=True):
      __tablename__ = "diagnosis"
      id: int | None = Field(default=None, primary_key=True)
      patient_id: int = Field(foreign_key="patient.id", index=True)
      name: str
      diagnosed_at: date
      status: str = "active"
  # Visit / Medication / LabResult / Billing 도 동일한 형태(patient_id FK 포함)
  ```
- **나이 계산(birth_date → age):**
  ```python
  from datetime import date
  def calc_age(birth: date, today: date | None = None) -> int:
      today = today or date.today()
      return today.year - birth.year - ((today.month, today.day) < (birth.month, birth.day))
  ```
- **보호된 목록 엔드포인트:**
  ```python
  from .models import Patient, Visit, Diagnosis, Medication, LabResult, Billing
  from .security import get_current_user

  @app.get("/api/patients")
  def list_patients(
      current: Staff = Depends(get_current_user),
      session: Session = Depends(get_session),
  ) -> list[dict]:
      patients = session.exec(select(Patient)).all()
      return [
          {"id": p.id, "registration_number": p.registration_number,
           "name": p.name, "age": calc_age(p.birth_date),
           "gender": p.gender, "current_stage": p.current_stage}
          for p in patients
      ]
  ```
- **보호된 상세 엔드포인트(없으면 404):**
  ```python
  @app.get("/api/patients/{patient_id}")
  def get_patient(
      patient_id: int,
      current: Staff = Depends(get_current_user),
      session: Session = Depends(get_session),
  ) -> dict:
      patient = session.get(Patient, patient_id)
      if patient is None:
          raise HTTPException(status_code=404, detail="환자를 찾을 수 없습니다")
      def by_pid(model):
          return list(session.exec(select(model).where(model.patient_id == patient_id)).all())
      return {
          "patient": {
              "id": patient.id, "registration_number": patient.registration_number,
              "name": patient.name, "age": calc_age(patient.birth_date),
              "gender": patient.gender, "allergies": patient.allergies,
              "current_stage": patient.current_stage,
          },
          "visits": by_pid(Visit),
          "diagnoses": by_pid(Diagnosis),
          "medications": by_pid(Medication),
          "lab_results": by_pid(LabResult),
          "billings": by_pid(Billing),
      }
  ```

### 흔한 실수 방지 (이것만은 피하기)
- ❌ 환자 엔드포인트에 `get_current_user` 빼먹기 → 누구나 환자 정보 열람(NFR3 위반). **반드시 보호.**
