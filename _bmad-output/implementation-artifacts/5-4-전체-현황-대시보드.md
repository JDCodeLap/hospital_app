# Story 5.4: 전체 현황 대시보드

Status: done

## Story

As a 병원 관리자,
I want 오늘 방문 환자 수·단계별 혼잡도·평균 대기시간을 한눈에 보기를,
So that 병원 운영 상황을 빠르게 파악한다. (FR13)

## Acceptance Criteria

1. **대시보드 진입** — 관리자 페이지(`/admin`)의 **"전체 현황" 카드가 활성 링크**가 되어 `/admin/dashboard`로 이동한다. 이 화면은 **관리자만**(NFR2) 접근한다.
2. **요약 숫자** — 대시보드에 **전체 환자 수·전체 직원 수·오늘 방문 환자 수·평균 대기시간(분)·대기 초과 인원**이 숫자 카드로 보인다.
3. **단계별 혼잡도** — **접수·진료·검사·수납** 각 단계에 현재 몇 명이 있는지 **숫자 + 막대그래프**로 보인다(어디가 붐비는지 한눈에). 비표준 단계가 있으면 "기타"로 묶는다.
4. **평균 대기시간 계산** — 현재 단계 진입 시각(`StageEntry`, Story 4.4)이 있는 환자들의 대기 분을 평균낸 값(정수)과, 기준(`STAGE_OVERDUE_MINUTES`, 기본 30분) 초과 인원 수를 보여준다. 진입 기록이 없는 환자는 평균 계산에서 제외한다.
4. **집계는 효율적으로** — 카운트는 **DB 집계(`func.count()`)**로 계산한다(전체 행을 메모리로 적재하지 않음 — 5.1 deferred 이행). 환자 수가 늘어도 안전.
6. **관리자 전용 + 보호** — 대시보드 API(`GET /api/admin/dashboard`)는 `get_current_admin`으로 보호 — 일반 직원 **403**, 미인증 **401**.
7. **상태 표시 + 새로고침** — 로딩 중("…")·오류(친절 안내)·정상 상태를 구분해 보여주고, **새로고침 버튼**(44px)으로 최신 수치를 다시 불러올 수 있다(401/403→로그아웃).
8. **빈/0 값 안전** — 환자·방문이 0이어도 화면이 깨지지 않는다(0명·"0분"·빈 막대 정상 표시). 오늘 방문이 0이어도 전체 환자 수 등 나머지는 그대로 보인다.

> **범위 밖(하지 말 것):**
> - **기준값(대기 초과 시간 등) 설정 = Story 5.5(FR14)** — 이 화면은 `STAGE_OVERDUE_MINUTES`(하드코딩)를 **읽기만** 한다.
> - **부서(과: 내과 등) 단위 세분화** — "혼잡도"는 **진행 단계(접수/진료/검사/수납)** 기준으로 구현한다(아래 설계 판단 #1). `Visit.department` 기반 과별 통계는 후속.
> - **기간/날짜 필터·차트 라이브러리·데이터 내보내기(CSV)·실시간 자동 갱신** — 범위 밖. 그래프는 **CSS 막대**로(새 의존성 0), 갱신은 **수동 새로고침**만.
> - **직원별 통계·접근 로그·추세(시계열)** — 후속.

    ```python
    @app.get("/api/admin/dashboard")
    def admin_dashboard(
        current: Staff = Depends(get_current_admin),  # 관리자만(NFR2)
        session: Session = Depends(get_session),
    ) -> dict[str, object]:
        """관리자 현황 대시보드(Story 5.4, FR13).

        오늘 방문 환자 수·단계별 혼잡도·평균 대기시간을 집계해 돌려준다.
        '혼잡도'는 진행 단계(접수/진료/검사/수납) 기준(설계 판단 #1).
        대기시간은 StageEntry 진입 시각 기준(4.4 재사용). 기준값은 5.5 전까지 하드코딩.
        """
        now = datetime.now()
        today = now.date()

        # 요약 수 — DB 집계(메모리 적재 X)
        patient_count = session.scalar(select(func.count()).select_from(Patient)) or 0
        staff_count = session.scalar(select(func.count()).select_from(Staff)) or 0

        # 오늘 방문 환자 수 — 오늘 날짜에 방문 기록이 있는 '서로 다른 환자' 수.
        # visited_at(datetime)이 [오늘 00:00, 내일 00:00) 범위인 방문의 distinct 환자.
        today_patient_count = session.scalar(
            select(func.count(func.distinct(Visit.patient_id))).where(
                Visit.visited_at >= datetime(today.year, today.month, today.day),
                Visit.visited_at < datetime(today.year, today.month, today.day) + timedelta(days=1),
            )
        ) or 0

        # 단계별 혼잡도 — current_stage GROUP BY. 표준 4단계는 0이라도 포함, 그 외는 '기타'.
        rows = session.exec(
            select(Patient.current_stage, func.count()).group_by(Patient.current_stage)
        ).all()
        counts = {stage: int(n) for stage, n in rows}
        stage_distribution = [
            {"stage": s, "count": counts.pop(s, 0)} for s in STAGE_ORDER
        ]
        etc = sum(counts.values())  # 비표준 단계 합
        if etc:
            stage_distribution.append({"stage": "기타", "count": etc})

        # 평균 대기시간·초과 인원 — StageEntry(환자당 1건, 소규모) 기준. stage_wait_info 재사용.
        entries = session.exec(select(StageEntry)).all()
        waits = []
        overdue_count = 0
        for e in entries:
            minutes, is_overdue = stage_wait_info(e.entered_at, now)
            if minutes is not None:
                waits.append(minutes)
                if is_overdue:
                    overdue_count += 1
        avg_wait_minutes = round(sum(waits) / len(waits)) if waits else 0

        return {
            "patient_count": patient_count,
            "staff_count": staff_count,
            "today_patient_count": today_patient_count,
            "stage_distribution": stage_distribution,  # [{stage, count}] 접수→수납(+기타)
            "avg_wait_minutes": avg_wait_minutes,
            "overdue_count": overdue_count,
            "overdue_threshold_minutes": STAGE_OVERDUE_MINUTES,  # 표시용(5.5 전 하드코딩)
        }
    ```
  - [x] ★ `StageEntry`는 환자당 1건이라 전체 로드해도 소규모(집계 메모리 우려 없음). 카운트류만 `func.count()`로 처리.

## Dev Notes

### 이 스토리의 핵심 (한 줄)
**관리자용 현황 대시보드.** 새 `GET /api/admin/dashboard`(집계 전용, `get_current_admin` 보호)가 환자·직원 수·오늘 방문·단계별 혼잡도·평균 대기·초과 인원을 `func.count()`/`group_by`로 계산하고, `/admin/dashboard` 화면이 숫자 카드 + CSS 막대로 보여준다. 5.1 `/admin` 랜딩의 "전체 현황" 카드를 활성화.

### ⚠️ 이 스토리의 가장 중요한 설계 판단

1. **"과별 혼잡도" = 진행 단계(접수/진료/검사/수납) 기준으로 구현.**
   - `Patient.current_stage`는 환자가 **지금 어디 있는지**(실시간 위치) → "어디가 붐비나"(혼잡도)에 정확. 흐름판(4.3)·대기초과(4.4)와 일관.
   - `Visit.department`(내과 등)는 **방문 이력**이라 한 환자가 여러 과를 거쳐 "현재 혼잡도"로 쓰기 모호 → **단계 기반 채택**, 과(department)별 세분화는 후속(범위 밖).

2. **집계는 DB에서(`func.count()`), 행 로딩 금지.** 5.1 `admin_overview`가 `len(select().all())`로 전 행(해시·PHI 포함)을 메모리에 적재하던 것(5.1 deferred)을 이번에 **`func.count()`로 교체**하고, 새 대시보드도 같은 방식. 단 `StageEntry`(환자당 1건)는 소규모라 평균 계산용으로 전체 로드 허용(카운트는 func).

3. **그래프 = CSS 막대(새 의존성 0).** epic의 "숫자/그래프"는 차트 라이브러리 없이 `div` 너비 비율 막대로 충족. 색+숫자 병행(UX-DR8). 번들·복잡도 최소화(프로젝트 철학).

4. **읽기 전용 + 기준값 하드코딩.** 대시보드는 집계를 **읽기만** 한다(쓰기 0). `STAGE_OVERDUE_MINUTES`는 표시·판정에 쓰되 **수정 기능은 5.5**(FR14). 응답에 `overdue_threshold_minutes`를 같이 내려 화면이 "기준 N분 초과 M명"처럼 표시할 수 있게.

5. **관리자 전용은 백엔드가 강제.** `get_current_admin`(403) — AdminGuard·메뉴는 UX 보조(5.1/5.3 철학). RSC 프리렌더 금지(1.4) — AdminGuard 통과 후 클라이언트 fetch.
