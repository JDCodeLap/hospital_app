## 5. 액션 아이템 (담당·기준)

### A. 이해·방향 돕기 (YC가 꼽은 최대 마찰 — 최우선)
- **A1.** 매 작업 시작/끝에 **"현 위치 지도"**(전체 중 어디 / 방금 한 일 / 다음 할 일)를 항상 한 줄로 제시. — 담당: Claude — 기준: 모든 응답에 적용.
- **A2.** 새 용어가 나오면 **즉시 괄호 풀이 + "왜 필요한지" 한 줄.** — 담당: Claude — 기준: CLAUDE.md 설명 규칙 재확약.
- **A3.** 요청 시 **"진행 현황 한 장"**(sprint-status를 쉬운 말로 요약) 제공. — 담당: Claude.

### B. 자동 테스트 도입 (YC가 꼽은 고칠 것)
- **B1.** **핵심 안전 로직부터** 자동 테스트 도입 검토 — 우선순위: ①알레르기/금기 차단(safety.py) ②권한 게이트(403/401) ③대기 초과 판정. 도구: 백엔드 pytest(+ `bmad-qa-generate-e2e-tests` 스킬 활용 가능). — 담당: Claude+YC — 시점: 5.5 완료 후 또는 Epic 6 전 작은 스파이크. — 기준: "절대 깨지면 안 되는 것"부터 통과 테스트 확보.
  - 비유: "물 새면 자동으로 울리는 센서"를 핵심 배관부터 달기.

### C. 기술 부채 — "배포 결심" 시 일괄 (지금 데모엔 무해)
- **C1.** naive datetime → **UTC(aware) 통일** (가장 load-bearing, #1).
- **C2.** 멀티워커 시드/백필 **동시성 가드**(try/except IntegrityError) (#3).
- **C3.** 오류 문구 **localhost 하드코딩 제거 + API base/HTTPS 필수화** (배포 전).
- **C4.** (선택) 감사 로그(누가 위험 처방·메모 수정) — Epic 5 권한/감사와 함께 (#5).

### D. 운영 습관
- **D1.** 커밋/푸시를 **작업(스토리 done) 단위로 더 자주.** (오늘 8커밋 쌓였던 것 해소 — 재발 방지.)
- **D2.** **에픽 끝날 때마다 회고**(이번이 첫 회고 — 습관화).
